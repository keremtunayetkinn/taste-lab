# Taste-Lab 🍳

An AI-powered recipe suggestion app. Enter the ingredients you have at home and get 2–3 original recipes from Turkish and world cuisines — complete with photos, step-by-step instructions, and a flavor profile chart.

**Live demo:** [taste-lab-kerem-tuna-s-projects.vercel.app](https://taste-lab-kerem-tuna-s-projects.vercel.app)

---

## Features

- **AI recipe generation** — powered by Claude Haiku; suggests 2–3 recipes tailored to your ingredients and filters
- **Smart filters** — narrow results by cooking time, budget, servings, and difficulty
- **Recipe photos** — automatically fetched from Pexels based on each recipe
- **Flavor profile chart** — radar chart visualizing spice, sourness, saltiness, sweetness, umami, and bitterness
- **Favorites** — save recipes to local storage and revisit them anytime
- **Search history** — browse and re-open previous searches
- **Deep links** — share any recipe via a unique URL; works on WhatsApp, X, and Instagram
- **Bilingual** — full Turkish and English support with one-click language toggle
- **Response caching** — identical searches are served from Redis cache, no repeated API billing

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | Vanilla JS (SPA), Chart.js |
| Backend | Vercel Serverless Functions (Node.js) |
| AI | Anthropic Claude Haiku (`claude-haiku-4-5-20251001`) |
| Photos | Pexels API |
| Rate limiting & Cache | Upstash Redis |
| Hosting | Vercel |

## Getting Started

### Prerequisites

- [Vercel](https://vercel.com) account
- [Anthropic](https://console.anthropic.com) API key
- [Pexels](https://www.pexels.com/api) API key
- [Upstash](https://console.upstash.com) Redis database (free tier is sufficient)

### Local Development

```bash
# Install dependencies
npm install

# Create a local environment file
cp .env.example .env.local
# Fill in the values in .env.local

# Start the dev server
vercel dev
```

### Environment Variables

Set these in your Vercel project under **Settings → Environment Variables**:

| Variable | Description |
|---|---|
| `CLAUDE_API_KEY` | Anthropic API key |
| `PEXELS_API_KEY` | Pexels API key |
| `UPSTASH_REDIS_REST_URL` | Upstash Redis REST URL |
| `UPSTASH_REDIS_REST_TOKEN` | Upstash Redis REST token |

### Deploy

```bash
vercel --prod
```

## API Endpoints

Both endpoints are server-side only and protected by origin validation, CSRF header checks, and Upstash rate limiting.

### `POST /api/claude`

Generates recipe suggestions.

**Rate limit:** 10 requests / hour per IP

**Request:**
```json
{
  "messages": [{ "role": "user", "content": "Elimdeki malzemeler: domates, yumurta" }],
  "dil": "tr"
}
```

**Response:**
```json
{
  "tarifler": [
    {
      "ad": "Menemen",
      "aciklama": "Klasik Türk kahvaltısı...",
      "sure": "15 dakika",
      "zorluk": "Kolay",
      "kisi": 2,
      "malzemeler": ["2 domates", "3 yumurta"],
      "adimlar": ["Domatesleri doğrayın...", "..."],
      "tat_profili": { "aci": 2, "eksi": 1, "tuzlu": 5, "tatli": 3, "umami": 6, "bitter": 0 },
      "foto": "https://images.pexels.com/..."
    }
  ]
}
```

### `POST /api/pexels`

Fetches a recipe photo by search query (used as fallback).

**Rate limit:** 30 requests / minute per IP

## Security

- **Content Security Policy** — strict CSP with explicit allowlists for all external resources
- **XSS prevention** — all user-facing dynamic content is HTML-escaped via `esc()`
- **Prompt injection protection** — injection keyword stripping + character allowlist on all user input before it reaches the AI
- **Input validation** — server-side validation on all request fields including language, message length, and content size
- **API key protection** — `pexels_arama` (the optimized Pexels search term generated by Claude) is stripped server-side before the response reaches the client, so neither API key nor search strategy is exposed
- **Distributed rate limiting** — Upstash Redis sliding window prevents abuse across all serverless instances
- **CSRF protection** — `X-Requested-With: XMLHttpRequest` header required on all API calls
- **Origin allowlist** — requests from unknown origins are rejected at the edge
